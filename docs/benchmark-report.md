# Benchmark Report

## 실험 목적

단일 재고 row에 동시 요청이 집중되는 핫스팟 상황에서 Optimistic Lock과 Pessimistic Lock의 성능 특성을 비교한다. 핵심 관찰 대상은 처리량, 성공 요청 지연시간, tail latency, 재시도 비용이다.

## 공통 조건

- 초기 재고: `20000`
- 목표 성공 건수: `20000`
- Backoff: `5ms`
- 성공당 최대 재시도: `500`
- Phase 순서: `optimistic -> reset -> pessimistic`
- 동시성: `5 / 10 / 30 / 50 / 100`

## 해석

낮은 동시성에서는 낙관락의 재시도 비용이 제한적이지만, 동시성이 30 이상으로 올라가면 단일 row 충돌 확률이 급격히 커진다. 이때 낙관락은 업데이트 실패, 재조회, backoff가 누적되며 처리량과 tail latency가 함께 악화된다.

비관락은 요청을 DB row lock 대기로 직렬화하므로 재시도 폭발은 발생하지 않는다. 단일 row 핫스팟이라는 제한된 조건에서는 비관락이 더 안정적인 처리량과 지연시간을 보였다.

## 한계

이 결과는 단일 상품에 트래픽이 집중되는 상황을 모델링한다. 상품이 여러 row로 분산되는 일반 트래픽에서는 낙관락이 더 유리한 구간이 있을 수 있다. DB 리소스, HikariCP 풀 크기, 트랜잭션 격리 수준, 애플리케이션 스레드 설정에 따라 수치는 달라질 수 있다.
